# Audit: minmaxing System Gaps — 2026-04-21

## Critical Finding: /workflow is a Router, Not an Executor

The routing table describes skill chains but `/workflow` does NOT actively execute them. Skills are dead ends.

### Routing Chain is Broken

```
/workflow routing says:
  "build X" → /autoplan → /sprint → /verify → /ship

Reality:
  /autoplan → creates SPEC.md → DONE (no /sprint call)
  /sprint → runs agents → DONE (no /verify call)
  /verify → ACCEPT/REJECT → DONE (no /ship call)
  /ship → ships → DONE
```

### /workflow Must Be the Orchestrator

/workflow should:
1. Route to first skill
2. Collect output
3. Route to next skill in chain
4. Repeat until /ship or /review
5. NOT just describe the chain — EXECUTE it
