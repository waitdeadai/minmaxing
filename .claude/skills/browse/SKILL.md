# /browse

Backward-compatible research alias for older prompts and muscle memory.

**MAX_PARALLEL_AGENTS** still applies here as a ceiling, not a quota. Keep the work effectiveness-first.

Use `/browse` like this:
- route to `/webresearch` for narrow current web / docs / API verification
- route to `/deepresearch` for multi-branch, deeper, or explicitly high-quality investigation work

This alias exists so older workflows do not break while the repo standardizes on the stronger command names.

## Routing Rule

Choose `/webresearch` when the task is:
- current API / SDK / version / error verification
- a small number of distinct tracks
- primarily web / docs lookup

Choose `/deepresearch` when the task is:
- high-stakes or broad
- multi-branch
- adversarial or due-diligence oriented
- explicitly asking for deep research / investigation quality

## Alias Contract

Even through `/browse`, preserve the same core behavior:
- collaborative research plan
- distinct tracks only
- iterative `search -> read -> refine`
- source ledger when external facts matter
- follow-up research when the first pass is not enough

## Compatibility Note

Prefer the repo-owned names in new docs, prompts, and examples:
- `/deepresearch`
- `/webresearch`

Keep `/browse` working for compatibility, not as the primary identity of the system.
